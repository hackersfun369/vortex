Documentation

Everything you need to use Vortex — from quick start to full API reference.

Overview

Vortex exposes your local server to the internet via a named subdomain. Traffic flows through Cloudflare's network — Vortex never touches your payload.

localhost:3000  ←── Cloudflare Tunnel ───  myapp.hackersfun369.github.io
                                                        ↑
                                               visitor's HTTPS request

Installation

curl -fsSL https://hackersfun369.github.io/install.sh | sh

Installs cloudflared and the vortex binary. Supports Linux (amd64/arm64/arm), macOS (amd64/arm64), and Windows (amd64).

No install? Use the portal to get a one-command copy-paste.

Quick start

vortex 3000                        # random subdomain
vortex 3000 -s myapp               # custom subdomain → myapp.hackersfun369.github.io
vortex 3000 --privacy              # private, not listed
vortex 3000 -s myapp --reserve     # permanent reservation (API key required)

CLI reference

Flags

REST API

Base URL: https://hackersfun369.github.io

All responses are JSON. All request bodies are JSON with Content-Type: application/json.

POST/tunnel/create optional auth

Register a new tunnel. Guests can create non-reserved tunnels. API key required for reservation.

{
  "subdomain": "myapp",    // optional — auto-generates if omitted
  "port": 3000,            // optional — informational
  "privacy": false,        // optional — default false
  "reserve": true          // optional — requires Authorization header
}

Headers: Authorization: Bearer vrtx_yourkey (for reserve), X-Vortex-Privacy: 1 (alt to body privacy)

{
  "ok": true,
  "subdomain": "myapp",
  "url": "https://myapp.hackersfun369.github.io",
  "token": "save-this-shown-once",
  "reserved": true,
  "private": false,
  "created_at": "2025-01-01T00:00:00Z",
  "expires_after_inactivity": "30 days",
  "grace_period_if_reserved": "30 days",
  "cloudflared_cmd": "cloudflared tunnel --url http://localhost:3000 --hostname myapp.hackersfun369.github.io",
  "install_cmd": "curl -fsSL https://hackersfun369.github.io/install.sh | sh"
}

GET/tunnel/:subdomain

Get public info about a tunnel. Never returns token. Returns 404 for private tunnels.

{
  "subdomain": "myapp",
  "url": "https://myapp.hackersfun369.github.io",
  "reserved": true,
  "private": false,
  "status": "active",
  "last_seen": "2025-01-15T08:00:00Z",
  "created_at": "2025-01-01T00:00:00Z",
  "grace_until": null
}

Status values: active / inactive / grace / expired

DELETE/tunnel/:subdomain token required

Delete a tunnel and immediately free the subdomain.

Header: Authorization: Bearer <your-token>

{ "ok": true, "subdomain": "myapp", "message": "tunnel deleted, subdomain freed" }

GET/tunnels

List all active non-private tunnels.

{ "ok": true, "count": 3, "tunnels": [...], "updated_at": "2025-01-15T08:00:00Z" }

POST/tunnel/heartbeat/:subdomain token required

Update last_seen. Max 1 write per hour per subdomain. Called automatically by the CLI.

{ "ok": true, "subdomain": "myapp", "last_seen": "2025-01-15T10:00:00Z" }

GET/my/tunnels auth required

List all tunnels owned by the authenticated user.

Header: Authorization: Bearer vrtx_yourkey or session cookie

POST/my/apply-api-access auth required

Apply for direct API access. Owner reviews and approves manually.

{ "reason": "building a mobile app that creates tunnels on demand" }

Subdomain lifecycle

AVAILABLE ──register──► ACTIVE
                            │
                   30 days no heartbeat
                            │
                         EXPIRED
                            │
             ┌──────────────┴──────────────┐
             │                             │
        reserved?                    not reserved?
             │                             │
     30-day grace period          FREED IMMEDIATELY
             │
       grace expires
             │
     FREED — anyone can claim
  • Heartbeat sent automatically by CLI every 60 minutes while tunnel is running
  • Inactivity timeout: 30 days (owner-configurable)
  • Grace period (reserved): 30 days — only token holder can reclaim during this window
  • On expiry: visitor is redirected to /expired?sub=myapp

Privacy mode

Pass --privacy (CLI) or "privacy": true (API):

  • Tunnel Gist is stored as secret (not publicly listed on GitHub)
  • Excluded from GET /tunnels listing
  • Not shown on the live board
  • Only the subdomain name and token hash are stored — no metadata

Authentication

GitHub OAuth — click "Sign in" to authenticate. A personal API key (vrtx_...) is issued once and shown in your dashboard.

API key — send as Authorization: Bearer vrtx_yourkey. Required for reservation and /my/tunnels. API users must be approved by the owner.

API access — apply via the dashboard. Owner approves manually.
